# Regras do Projeto

- Repositorio GitHub principal: `ghcastellano/ProjetosDanilo`.
- Ao concluir uma tarefa de codigo neste projeto, publicar as alteracoes relevantes no GitHub em background quando houver acesso ao conector GitHub.
- Fazer commit com mensagem curta e clara descrevendo a alteracao.
- Nao publicar arquivos temporarios, locais ou auxiliares que nao facam parte do app, como scripts de servidor local, caches, credenciais ou arquivos com tokens.
- Antes de publicar, conferir rapidamente os arquivos alterados para evitar subir mudancas quebradas ou fora do escopo.
- Se `git` ou `gh` nao estiverem disponiveis no Windows, usar o conector GitHub para criar ou atualizar os arquivos no repositorio.
